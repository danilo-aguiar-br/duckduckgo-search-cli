### Troubleshooting

1. **HTTP 202 / block anomaly (exit 3)** вЂ” back off, raise `--retries`, rotate UA via `init-config` and tweak `user-agents.toml`.
2. **Rate limited (HTTP 429)** вЂ” lower `--per-host-limit`, enable `--match-platform-ua`, or add `--proxy`.
3. **Zero results (exit 5)** вЂ” check `--lang` and `--country`, verify `--time-filter`, ensure Chrome is usable (Lite is not a remediation path since GAP-WS-113).
4. **`--output` rejects my path (exit 2)** вЂ” v0.5.0 validates output paths before writing. Paths containing `..` are rejected to prevent directory traversal.
5. **Getting exit 5 (zero results) frequently** вЂ” wait 60 seconds and retry. Add `--proxy socks5://127.0.0.1:9050` to rotate outbound IP. Confirm Chrome (`--probe` / `--chrome-path`); do not use `--allow-lite-fallback` (no-op GAP-WS-113).

# How to Use duckduckgo-search-cli

## Why This Guide
- Follow this guide and run your first web search in under 60 seconds

### Proxy Routing
```bash

## Common Errors
### HTTP 202 Anti-bot Block (exit 3)
- DuckDuckGo returned a soft challenge page, not real results
- Wait 60 seconds before retrying
- Rotate outbound IP with `--proxy socks5://127.0.0.1:9050`
- Increase retries: `--retries 5`
- Run `duckduckgo-search-cli init-config` to refresh browser profiles
### Global Timeout (exit 4)
- Pipeline exceeded `--global-timeout` (default: 60 seconds)
- Increase value: `--global-timeout 120`
- Reduce result count: `-n 5`
- Aumente `--timeout` / `--global-timeout` em conexГµes lentas; mantenha SERP HTML Chrome
### Zero Results (exit 5)
- Often temporary rate-limiting, not a permanent block
- Wait 60 seconds and retry the same query
- Broaden the query by removing specific terms
- Remove `--time-filter` if set вЂ” it narrows the result pool
- Use `--proxy` / `--chrome-path` / aguarde-e-retente вЂ” nГЈo fallback Lite (GAP-WS-113)
### Invalid Config (exit 2)
- A flag is out of range or a path is invalid
- `--timeout 0` is rejected вЂ” minimum is 1 second
- `--output ../../../etc/passwd` is rejected вЂ” path traversal blocked
- `--global-timeout 0` is rejected вЂ” minimum is 1 second
- `--parallel 0` is rejected вЂ” minimum is 1


## Exit Codes Reference

| Code | Meaning | Recommended Action |
|------|---------|-------------------|
| 0 | Success | Process results normally |
| 1 | Runtime error (network, parse, I/O) | Check stderr for details |
| 2 | Invalid config (flag out of range, bad path) | Fix the argument |
| 3 | DuckDuckGo anti-bot block (HTTP 202) | Wait 60s or rotate proxy |
| 4 | Global timeout exceeded | Increase `--global-timeout` |
| 5 | Zero results across all queries | Broaden query or remove filters |

ddg-deep() {

    echo "usage: ddg-deep <query...>" >&2

  {
    "query": "...", "motor": "duckduckgo", "endpoint": "html",
    "timestamp": "2026-04-14T10:00:00Z", "regiao": "br-pt",
    "quantidade_resultados": 15, "paginas_buscadas": 2,
    "resultados": [
      {"posicao": 1, "titulo": "...", "url": "...", "snippet": "...", "url_exibicao": "...", "titulo_original": "..."}
    ],
    "metadados": {"tempo_execucao_ms": 1234, "user_agent": "..."}
  }
  ```
- Pipe safety: SIGPIPE restored to SIG_DFL on Unix вЂ” pipes terminate cleanly. BrokenPipe returns exit 0.
- Path safety (v0.5.0): `--output` validates paths BEFORE writing вЂ” rejects `..` components and system directories.
- Credential safety (v0.5.0): proxy credentials NEVER appear in error messages вЂ” automatic masking.
- Anti-blocking (v0.6.0): `BrowserProfile` injects per-family `Sec-Fetch-*`, Client Hints, and RFC 7231 `Accept-Language`.
- Multi-query schema: `{quantidade_queries, timestamp, paralelismo, buscas: [<SingleSchema>]}`
